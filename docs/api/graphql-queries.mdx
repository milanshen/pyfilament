Pyfilament's GraphQL API exposes seven query fields for retrieving task types and task runs. Use these to monitor task execution, inspect results, and build custom dashboards.

See the [GraphQL Overview](/api/graphql-overview) for a description of all returned types.

---

## getTaskRun

Fetch a single task run by its database ID or UUID.

**Arguments:**

<ParamField query="id" type="ID">
  The database ID of the task run.
</ParamField>

<ParamField query="taskUuid" type="String">
  The UUID of the task run.
</ParamField>

Provide either `id` or `taskUuid` — at least one is required.

**Returns:** `TaskRun`

```graphql
query {
  getTaskRun(taskUuid: "abc123") {
    id
    taskUuid
    state
    parametersJson
    resultJson
    logs(withChildren: true, maxDepth: 3, maxNumChildren: 100) {
      timestamp
      level
      message
    }
  }
}
```

The `logs` field accepts optional arguments to control how child task logs are collected:

| Argument | Type | Default | Description |
|---|---|---|---|
| `withChildren` | `Boolean` | `true` | Whether to include logs from child tasks. |
| `maxDepth` | `Int` | `3` | How many levels of child tasks to traverse for logs. |
| `maxNumChildren` | `Int` | `100` | Maximum number of children to collect logs from per level. |

## getTaskType

Fetch a single task type by its database ID or `funcAddress`.

**Arguments:**

<ParamField query="id" type="ID">
  The database ID of the task type.
</ParamField>

<ParamField query="funcAddress" type="String">
  The module path and function name of the task, e.g. `myapp.tasks:process_order`.
</ParamField>

Provide either `id` or `funcAddress` — at least one is required.

**Returns:** `TaskType`

```graphql
query {
  getTaskType(funcAddress: "myapp.tasks:process_order") {
    id
    name
    parametersSpec
    resultSpec
  }
}
```

## getTaskTypes

List all task types that have had at least one run within the last N days.

**Arguments:**

<ParamField query="days" type="Int" default="3">
  How many days of history to include. Task types with no runs in this window are excluded.
</ParamField>

**Returns:** `[TaskType]`

```graphql
query {
  getTaskTypes(days: 7) {
    id
    name
    latestTaskRun {
      state
      stateSince
    }
  }
}
```

## getTaskRuns

List task runs for a specific task type, optionally filtered by state and time window. Results are ordered newest first and are limited to 99 runs.

**Arguments:**

<ParamField query="taskTypeId" type="ID" required>
  The database ID of the task type.
</ParamField>

<ParamField query="states" type="[String]">
  Filter to runs in these states only. Values must match [TaskState](/api/task-states) strings, e.g. `["running", "failure"]`. Omit to return all states.
</ParamField>

<ParamField query="days" type="Int" default="3">
  How many days of history to include.
</ParamField>

**Returns:** `[TaskRun]`

```graphql
query {
  getTaskRuns(taskTypeId: "1", states: ["failure", "timeout"]) {
    taskUuid
    state
    createdAt
    runCount
  }
}
```

## getTaskRunsByIds

Fetch multiple task runs by a list of database IDs. Returns runs in the same order as the input IDs. Returns a `404 Not Found` error if any ID does not exist.

**Arguments:**

<ParamField query="ids" type="[Int]" required>
  List of task run database IDs to fetch.
</ParamField>

**Returns:** `[TaskRun]`

```graphql
query {
  getTaskRunsByIds(ids: [10, 11, 12]) {
    id
    taskUuid
    state
    createdAt
  }
}
```

---

## getTaskTypesByIds

Fetch multiple task types by a list of database IDs or UUIDs. Returns only the task types that were found; missing IDs are silently omitted. When `ids` are provided the result order matches the input order.

**Arguments:**

<ParamField query="ids" type="[Int]">
  List of task type database IDs to fetch.
</ParamField>

<ParamField query="uuids" type="[String]">
  List of task type UUIDs to fetch.
</ParamField>

Provide either `ids` or `uuids` — at least one is required.

**Returns:** `[TaskType]`

```graphql
query {
  getTaskTypesByIds(ids: [1, 2, 3]) {
    id
    name
    funcAddress
  }
}
```

## getTaskTypeStackRuns

Find leaf task runs that belong to a specific nested call chain. Supply an ordered list of task type IDs representing the call stack from root to leaf; Pyfilament returns runs matching the deepest task type whose ancestors match the full chain.

Results are limited to 100 runs, ordered newest first.

**Arguments:**

<ParamField query="taskTypeIds" type="[Int]" required>
  Ordered list of task type IDs from root to leaf, e.g. `[1, 3, 7]` to find runs of task type 7 that were spawned by task type 3, which was spawned by task type 1.
</ParamField>

<ParamField query="states" type="[String]">
  Filter leaf runs by state. Omit to return all states.
</ParamField>

**Returns:** `[TaskRun]`

```graphql
query {
  getTaskTypeStackRuns(taskTypeIds: [1, 3, 7], states: ["failure"]) {
    taskUuid
    state
    stateSince
    parentTaskUuid
  }
}
```

**Use case:** use this query to find all leaf task runs in a nested call chain — for example, to pinpoint which subtask in a multi-step pipeline is failing.
