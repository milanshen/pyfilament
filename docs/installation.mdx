Pyfilament is published to [PyPI](https://pypi.org/project/pyfilament/) and installs with a single command. It requires **Python 3.14 or higher**. Redis is required for distributed workers, result caching, and the observability dashboard — but you can run local tasks without it.

## Install Pyfilament

<CodeGroup>
  ```bash uv
  uv add pyfilament
  ```

  ```bash pip
  pip install pyfilament
  ```
</CodeGroup>

<Tip>
  Use `uv` for the fastest install experience. `uv` manages Python versions automatically, so it will download and pin Python 3.14 for your project if you do not already have it.
</Tip>

## Requirements

- **Python 3.14 or higher** — Pyfilament targets CPython 3.14 and will not install on earlier versions.
- **Redis 7+** — Required for distributed workers, result caching, rate limiting, and the observability dashboard.

<Note>
  Local-only tasks — those without distributed workers or caching — work without Redis. You only need Redis when you enable `cache=True`, `rate_limit`, `max_concurrent` with remote workers, or the observability server.
</Note>

## Configure Redis

Pyfilament connects to Redis at `localhost:6379` on database `0` by default. If your Redis runs there, no configuration is needed.

To point at a different Redis, set any of these environment variables in your shell or a `.env` file at the root of your project — Pyfilament loads `.env` automatically at startup. Each falls back to its default when unset, so you only set the ones you need to change:

```ini .env
FILAMENT_REDIS_HOST=redis.internal   # default: localhost
FILAMENT_REDIS_PORT=6380             # default: 6379
FILAMENT_REDIS_DB=1                  # default: 0
```

## Configure the database

Pyfilament records every task run in a database. By default it uses a local SQLite file (`filament.sqlite`) created in your working directory, so no configuration is needed to get started.

To use Postgres instead, set `FILAMENT_DB_URI` to an async connection string:

```ini .env
FILAMENT_DB_URI=postgresql+asyncpg://user:pass@localhost:5432/filament
```

Both the workers running your tasks and the observability dashboard read this database, so point them at the same one.

### Create the schema

Pyfilament does not create its tables automatically. Apply the Alembic migrations once against your database before running any task — every task run records its state there, and the dashboard reads the same tables:

```bash
uv run alembic upgrade head
```

<Note>
  The migrations live in the [Pyfilament repository](https://github.com/pyfilament/pyfilament) and are run from a checkout of it; they are not bundled in the published package. Re-run the command after pulling changes that add new migrations.
</Note>

## Verify your installation

Create a small script to confirm everything is wired up correctly:

```python verify.py
from filament import task
from asyncio import run

@task
async def ping() -> str:
    return "pong"

result = run(ping())
print(result)  # pong
```

Run it:

```bash
python verify.py
```

You should see `pong` printed to stdout. If you see an import error, confirm that your virtual environment is active and that `pyfilament` is listed in your installed packages.

## Start the observability server

Pyfilament ships a **FastAPI** backend and a **React** frontend that give you a live dashboard for monitoring task runs: state timelines, structured logs, captured inputs and outputs, and worker status. Point both at the same database and Redis instance your tasks use.

See [Observability Dashboard](/observability/dashboard#start-the-dashboard) for the commands to start the backend and frontend.
